# PatchHub Project Docs

This folder is only for project-specific architecture that is easy to get wrong by reading code
in isolation.

- `auth-and-organizations.md` — WorkOS, Convex users, ownership, and the unfinished organization
  model.
- `remote-functions-guidelines.md` — how PatchHub uses SvelteKit remote functions, including its
  SSR status-code workaround.
- `scripts.md` — current development, validation, test, and Worker commands.
- `workspace-layout.md` — ownership boundaries for the main folders.

Do not add task plans, handoff files, generic TypeScript advice, or framework documentation here.
Prefer the code, tests, official framework documentation, and `AGENTS.md` for those concerns.
